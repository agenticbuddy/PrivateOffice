## Quality observations

- **Scripts render correctly** across Latin, Cyrillic, Greek-free CJK (Chinese,
  Japanese, Korean), Devanagari (Hindi), Thai, and RTL Semitic/Perso-Arabic (Arabic,
  Hebrew, Persian) — the system font stack (`Inter` + `Noto Sans` fallback) covers all.
- **RTL is fully mirrored**, not just text-aligned: the sidebar, top bar, action
  buttons and drawers flip side because the CSS uses logical properties
  (`margin-inline`, `inset-inline-*`, `border-inline-*`) and `dir` is set on `<html>`.
  Verified visually for `ar` (logo + nav on the right, avatar on the left).
- **No clipping or horizontal overflow** at desktop width for any locale, including the
  longer German/Dutch labels and the wide CJK button text.
- **Editor language**: the editor toolbar/menus and document language follow the
  viewer's locale via the `lang` URL parameter — verified live in Russian (xlsx) and
  English (docx).

## Translation conventions

- **Brand names are do-not-translate**: the product name `app.name` is always the
  verbatim brand `PrivateOffice` in every catalog (like font names, it is never
  localized). Only descriptive copy such as `app.tagline` is translated. When adding a
  locale, copy `app.name` from `en.json` verbatim rather than translating it.

## Known limitations / honest notes

- Number, date and relative-time formatting uses the browser `Intl` API with the app
  locale; a few locales (e.g. `pt-BR`, `zh-CN`) are passed through to `Intl` as-is and
  render correctly, but locale-specific calendar variants were not separately audited.
- Spellcheck dictionaries in the editor remain governed by the editor `allowed_languages`
  config (default set), independent of the UI language — see `docs/LANGUAGES.md`.
- Screenshots in `e2e/lang-sweep/` are the authoritative visual record.
